Getting Started with TACS.NET

Welcome to the TACS.NET Manager application.  TACS.NET is a client/server framework designed to provide access control services for web and desktop applications built on Microsoft's .NET Framework.  With this application you may manage TACS.NET applications, accounts, project databases, security roles, and user profiles.  Please refer to the TACS.NET Software Development Kit for additional documentation on building applications using this framework.

TACS.NET Objects

In order to understand TACS.NET's architecture, it is helpful to understand the various objects available and their functionality.  See the following definitions:

  • Application -- An application that uses this framework must be registered in the TACS.NET database.  Registered applications are shown in the Application Explorer window.

  • Account -- Project databases and user profiles are grouped under accounts.  An account represents a client with one or more project databases and user profiles.  An account can be opened from the File...Open menu option or by pressing CTRL+O.

  • Project -- Represents a project database.  The project record contains the database access credentials for a given application.  Credentials are generated when a user successfully authenticates with the TACS.NET Server, thus eliminating the need to store credentials on the user's local computer.  Projects registered under an account are displayed in the Project Explorer window after you open an account.

  • Roles -- Security roles are created on a per project basis.  Applications use role objects to decide what features a user has access to.  Roles are edited by opening the associated project from the Project Explorer.

  • Users -- User objects define which users are allowed to login and use specified projects with certain applications.  Users registered under an account are displayed in the User Explorer window after you open an account.  You may also assign users to projects by editing a specific user profile.
